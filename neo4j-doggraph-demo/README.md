# DogGraph — Behavioral Intelligence as a Graph (Neo4j)

![DogGraph schema](screenshots/doggraph_schema.png)

> Synthetic data · Research demonstrator · Not a diagnostic tool · Patent applications filed

**A minimal Neo4j demo of Barkley's core idea: a dog's behavioral intelligence is
fundamentally _relational_ — dog ↔ baseline ↔ context ↔ drift ↔ environment ↔
compatible dogs — so it belongs in a graph, not a flat table of snapshots.**

This is a focused hackathon add-on for the
[Proof of Usefulness](https://proofofusefulness.com) hackathon. It runs on
**Neo4j AuraDB** (free tier) with **synthetic data only**.

---

## Why a graph (and why this isn't "Neo4j for the sponsor badge")

Most pet tech stores flat measurements and compares each dog to a breed average.
Barkley measures each dog against **itself**, over time, in **context**. Every one
of those words is a relationship:

- *"Is this dog drifting from **its own** baseline?"* — a path: `Dog → DriftEvent → Baseline`.
- *"Is that change **explained by** something?"* — an edge: `DriftEvent → ContextEvent`.
- *"Which route fits its **current state**?"* — `Dog → Route → Location`.
- *"Which dog is it **compatible with**, and why?"* — `Dog → COMPATIBLE_WITH → Dog`.

A flat tracker cannot answer these without expensive joins and lost meaning. A graph
answers them natively. That's the honest fit: **DogGraph is the intelligence layer,
and a property graph is its natural shape.**

## The model

Nodes: `Dog`, `Baseline` (the Individual Cognitive Fingerprint), `TemporalBin`
(circadian / weekly / quarterly), `BehaviorEvent`, `ContextEvent` (the missingness /
"Sovereignty of Silence" taxonomy), `DriftEvent`, `Route`, `Location`.

Key relationships:

```
(:Dog)-[:HAS_BASELINE]->(:Baseline)
(:Dog)-[:GENERATED]->(:BehaviorEvent)-[:FALLS_IN]->(:TemporalBin)
(:Dog)-[:HAS_DRIFT]->(:DriftEvent)-[:DRIFTED_FROM]->(:Baseline)
(:DriftEvent)-[:MODULATED_BY]->(:ContextEvent)
(:Dog)-[:RECOMMENDED_ROUTE {reason}]->(:Route)-[:LOCATED_NEAR]->(:Location)
(:Dog)-[:COMPATIBLE_WITH {score, reason}]->(:Dog)
```

Full definitions and constraints are in [`schema.cypher`](./schema.cypher).

## Quickstart (Neo4j AuraDB free, ~5 minutes)

1. Create a free **AuraDB** instance at <https://neo4j.com/product/auradb/> and save
   the connection URI + password.
2. Open **Neo4j Browser** (or the Aura query console) and run, in order:
   ```cypher
   // 1) constraints + indexes
   :source schema.cypher          // or paste the file contents
   // 2) synthetic graph
   :source seed_synthetic.cypher  // ~50 nodes, fully synthetic
   // 3) the showcase questions
   :source queries.cypher
   ```
   (In the Aura console, paste each file's contents if `:source` isn't available.)
3. Run the **BONUS** query at the bottom of `queries.cypher` and take a screenshot of
   Kikoo's neighbourhood for your writeup.

`seed_synthetic.cypher` is generated by [`generate_seed.py`](./generate_seed.py) — re-run
it to regenerate or grow the synthetic graph.

## The four showcase queries

See [`queries.cypher`](./queries.cypher). Each answers a relational question:

1. **Which dogs are drifting from their own baseline?** — individual-referenced change,
   ranked by drift rate (not a population comparison).
2. **Which drifts are explained by context vs. unexplained?** — the Sovereignty of
   Silence in action: a context-explained change is calm; an unexplained one earns a look.
3. **Which route fits Kikoo's current state, and why?** — recommendation as a function
   of present behavioral state (elevated drift → calmer route).
4. **Which dog is socially compatible with Kikoo, and why?** — a scored, *explained*
   match relationship.

## Natural-language layer (`ask.py`)

A small natural-language → Cypher layer so you can ask the graph in plain English:

```bash
pip install -r requirements.txt

# ask.py reads these directly from the environment (see .env.example for the names)
export NEO4J_URI="neo4j+s://<your-aura-id>.databases.neo4j.io"
export NEO4J_USER="neo4j"
export NEO4J_PASSWORD="<your-aura-password>"

python ask.py "Is Kikoo's drift explained by context, or should I worry?"
python ask.py "Which dog is socially compatible with Kikoo?"
```

**Honest description:** this is a **deterministic intent router** that maps a question
to one of a handful of *audited, parameterised* Cypher queries — it does **not** use an
LLM and does not invent queries. The seam to upgrade to true LLM-backed,
schema-constrained text-to-Cypher ("GraphRAG") is marked in
[`graph_query.py`](./graph_query.py). Only claim the `#graphrag` /
`#graph-powered-agents` angle once that LLM step is actually wired in.

## What this enables

Once behavior lives in a graph, the Barkley primitives become queries, not bespoke
code: individual baselines, rate-of-drift detection, context-aware interpretation of
silence, state-aware route recommendation, and explained social compatibility — all
traversals over one connected behavioral memory.
