# DogGraph — Behavioral Intelligence as a Graph (Neo4j) · v2

![DogGraph seeded subgraph](screenshots/graph_render.png)

> Synthetic data · Research demonstrator · Not a diagnostic tool · Patent applications filed

🔗 **Live app:** **[barkley-doggraph.streamlit.app](https://barkley-doggraph.streamlit.app/)** — the contents of this folder, deployed against a real AuraDB instance, with the LLM/GraphRAG layer enabled.

**A focused Neo4j demo of Barkley's core idea: a dog's behavioral intelligence is
fundamentally _relational_ — dog ↔ baseline ↔ context ↔ drift ↔ environment ↔
compatible dogs — so it belongs in a graph, not a flat table of snapshots.**

Runs on **Neo4j AuraDB** (free tier) with **synthetic data only**, and ships with
three Python layers on top: a deterministic intent router, a real
**schema-constrained LLM text-to-Cypher** layer with **GraphRAG synthesis**, and a
**Streamlit app** to drive both from a browser.

## What changed in v2

- 🧠 **Schema-constrained GraphRAG layer** — `graph_query_llm.py` translates natural-language
  questions to **read-only** Cypher under a schema-constrained Claude prompt, runs
  it on Aura, and synthesizes a grounded answer from the rows. Two layers of safety
  (prompt-level rule + post-hoc validator that refuses `CREATE`/`MERGE`/`DELETE`/
  `SET`/`REMOVE`/`DROP`/`DETACH`/`LOAD CSV`/etc.).
- 🖥 **Full Streamlit app** — `app.py`, Aura-connected, with the LLM/GraphRAG layer
  behind the input box, schema sidebar, preset queries, and a results table.
  Deployable on Streamlit Community Cloud (free tier) in ~5 minutes.
- 🌐 **Publication-quality graph render** — `make_seed_render.py` produces
  `screenshots/graph_render.png` (above) directly from the seeded structure.
- ⌨️ **`ask.py --llm`** — same CLI, now with an opt-in LLM mode that goes through
  the GraphRAG pipeline.
- Honest fallback: with no `ANTHROPIC_API_KEY`, every path degrades gracefully to the
  v1 deterministic router. Nothing breaks.

## Why a graph (and why this isn't "Neo4j for the sponsor badge")

Most pet tech stores flat measurements and compares each dog to a breed average.
Barkley measures each dog against **itself**, over time, in **context**. Every one
of those words is a relationship:

- *"Is this dog drifting from **its own** baseline?"* — a path: `Dog → DriftEvent → Baseline`.
- *"Is that change **explained by** something?"* — an edge: `DriftEvent → ContextEvent`.
- *"Which route fits its **current state**?"* — `Dog → Route → Location`.
- *"Which dog is it **compatible with**, and why?"* — `Dog → COMPATIBLE_WITH → Dog`.

A flat tracker cannot answer these without expensive joins and lost meaning. A graph
answers them natively. That's the honest fit: **DogGraph is the intelligence layer,
and a property graph is its natural shape.**

## The model

![DogGraph schema](screenshots/doggraph_schema.png)

Nodes: `Dog`, `Baseline` (the Individual Cognitive Fingerprint), `TemporalBin`
(circadian / weekly / quarterly), `BehaviorEvent`, `ContextEvent` (the missingness /
"Sovereignty of Silence" taxonomy), `DriftEvent`, `Route`, `Location`.

Key relationships:

```
(:Dog)-[:HAS_BASELINE]->(:Baseline)
(:Dog)-[:GENERATED]->(:BehaviorEvent)-[:FALLS_IN]->(:TemporalBin)
(:Dog)-[:HAS_DRIFT]->(:DriftEvent)-[:DRIFTED_FROM]->(:Baseline)
(:DriftEvent)-[:MODULATED_BY]->(:ContextEvent)
(:Dog)-[:RECOMMENDED_ROUTE {reason}]->(:Route)-[:LOCATED_NEAR]->(:Location)
(:Dog)-[:COMPATIBLE_WITH {score, reason}]->(:Dog)
```

Full definitions and constraints are in [`schema.cypher`](./schema.cypher).

## Quickstart — the graph itself (~5 min)

1. Create a free **AuraDB** instance at <https://neo4j.com/product/auradb/> and save
   the connection URI + password.
2. Open **Neo4j Browser** (or the Aura query console) and run, in order:
   ```cypher
   // 1) constraints + indexes
   :source schema.cypher
   // 2) synthetic graph (~50 nodes, fully synthetic)
   :source seed_synthetic.cypher
   // 3) the showcase questions
   :source queries.cypher
   ```
3. Run the **bonus** visualization query at the bottom of `queries.cypher` and
   screenshot Kikoo's neighborhood — that's your authentic Browser/Bloom render
   for any writeup. (`screenshots/graph_render.png` is a programmatic equivalent.)

`seed_synthetic.cypher` is generated by [`generate_seed.py`](./generate_seed.py) —
re-run it to regenerate or grow the synthetic graph.

## Quickstart — Streamlit app (~5 min)

Run locally:

```bash
pip install -r requirements.txt

export NEO4J_URI="neo4j+s://<your-aura-id>.databases.neo4j.io"
export NEO4J_USER="neo4j"
export NEO4J_PASSWORD="<your-aura-password>"
export ANTHROPIC_API_KEY="<your-anthropic-key>"          # optional; enables GraphRAG

streamlit run app.py
```

Deploy on **Streamlit Community Cloud** (free):

1. Push this folder to a GitHub repo (it already lives in
   `labs-barkley/barkley-canine-cognition-lab/neo4j-doggraph-demo/`).
2. <https://share.streamlit.io> → **Create app** → select the repo, set the main
   file to `neo4j-doggraph-demo/app.py`.
3. **Advanced settings → Secrets**: paste the contents of
   [`.streamlit/secrets.toml.example`](./.streamlit/secrets.toml.example) with
   real values.
4. Deploy.

The free Community tier (2.7 GB per app, unlimited public apps) is more than enough
for this demo.

## The four showcase queries

See [`queries.cypher`](./queries.cypher). Each answers a relational question:

1. **Which dogs are drifting from their own baseline?** — individual-referenced
   change, ranked by drift rate (not a population comparison).
2. **Which drifts are explained by context vs. unexplained?** — the Sovereignty of
   Silence in action: a context-explained change is calm; an unexplained one earns
   a look.
3. **Which route fits Kikoo's current state, and why?** — recommendation as a
   function of present behavioral state (elevated drift → calmer route).
4. **Which dog is socially compatible with Kikoo, and why?** — a scored,
   *explained* match relationship.

## Natural-language layer

Two implementations sit behind the same CLI / Streamlit input:

### Deterministic intent router — `graph_query.py`

Maps a question to one of four audited, parameterised Cypher queries by keyword
pattern. Predictable, no LLM, no surprises. The right thing for sensitive demos.

```bash
python ask.py "Is Kikoo's drift explained by context?"
```

### LLM + GraphRAG — `graph_query_llm.py`

Real schema-constrained text-to-Cypher with grounded answer synthesis. The pipeline
is precisely:

```
question
  → Claude (system: full DogGraph schema, few-shots, read-only rule, OUT_OF_SCOPE escape)
  → Cypher
  → validate_readonly()                ← forbids CREATE/MERGE/DELETE/SET/REMOVE/DROP/...
  → execute on Neo4j AuraDB            ← retrieval = graph traversal
  → rows
  → Claude (system: "ground answer strictly in these rows")
  → natural-language answer
```

Two-layer safety: the system prompt states the read-only rule and the schema; the
post-hoc validator scans the emitted Cypher and **refuses** anything that contains
a write keyword, even if the LLM ignored its instructions. No write ever reaches
the database.

```bash
export ANTHROPIC_API_KEY="<your-key>"
python ask.py --llm "Which dog has the highest unexplained drift?"
```

If `ANTHROPIC_API_KEY` is unset, the LLM layer transparently falls back to the
deterministic router.

## What this enables

Once behavior lives in a graph, the Barkley primitives become queries, not bespoke
code: individual baselines, rate-of-drift detection, context-aware interpretation
of silence, state-aware route recommendation, and explained social compatibility —
all traversals over one connected behavioral memory. Adding the LLM/GraphRAG layer
makes this conversational without giving up auditability or read-only safety.
